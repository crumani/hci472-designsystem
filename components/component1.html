Navbar

A navbar holds navigation information at the top of a webpage.

Example:

  • Home
  • Research
  • Design
  • Resume

Code:


      <nav class="navbar navbar-expand-lg bg-nav sticky-top">
        <div class="container-fluid">
          <a class="navbar-brand link-light" href="#"><img src="../img/logo1.png" class="card-img-top" alt="Black and white logo"></a></a>
          <button class="navbar-toggler" type="button" data-bs-toggle="collapse"
            data-bs-target="#navbarNavDropdown" aria-controls="navbarNavDropdown" aria-expanded="false"
            aria-label="Toggle navigation"> 
            <span class="navbar-toggler-icon"></span>
          </button>
          <div class="collapse navbar-collapse" id="navbarNavDropdown">
            <ul class="navbar-nav">
              <li class="nav-item">
                <a class="nav-link active nav-highlight link-light" aria-current="page" href="#">Home</a>
              </li<
              <li class="nav-item">
                <a class="nav-link link-light" href="#">Research</a>
              </li<
              <li class="nav-item">
                <a class="nav-link link-light" href="#">Design</a>
              </li<
              <li class="nav-item">
                <a class="nav-link link-light" href="#">Resume</a>
              </li>
            </ul>
          </div>
        </div>
      </nav>

Usage:

The navbar must be included: on all webpages, at the top of the page, with the same items in the same order, and must link back to the main page

Accessibility:

Use semantic tags, such as <nav/>, and the ARIA role = "navigation" attribute. This ensures screenreaders and other accessibility tools can properly read the navigation bar. Ensure this is consistent across both mobile and desktop views.
